## Philosophy

Ship a **2-week UX sprint** that cuts cognitive load, clarifies two paths, puts trust at the moment of submit, and closes the loop after submit. Sequence the page so **menu appetite supports form completion**.

---

## Phase A actions (from DeepSeek strategy)

1. Cut required fields ~50% (org / event type / package optional)  
2. Native date/time pickers  
3. Two-path labels with **speed vs customization** trade-off  
4. Trust anchors near submit (only real stats/logos)  
5. Post-submit confirmation + response SLA + auto-reply expectation  
6. Menu preview positioned to support (not hide behind) the form  
7. Instrument: abandonment, path select, time-to-complete, submit success, follow-through  

---

## Explicitly later

- Full visual redesign of packages as the first move  
- Fake social proof  
- Multi-path menus beyond the two clear jobs  

---

## Vote: top 5

1. Shorter form (cognitive load)  
2. Native date/time  
3. Two-path trade-off labels  
4. Trust + confirmation at submit / post-submit  
5. Menu sequence that rewards completing the form
